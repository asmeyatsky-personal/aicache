# aicache refactor & build plan — 2026-04-20

Output of the planning session. Organised as seven phases aligned to `Architectural Rules — 2026.md`. Each phase is independently reviewable and commit-isolated.

## Guiding constraints

- **2026 rules as the rubric.** Every phase must advance compliance with §1–§7. Rules not enforced in CI don't count (§7.10).
- **Dogfood the hex architecture.** By end of plan, `modern_cli.py`, MCP server, and a new SDK client path all route through `application/use_cases.py`. No runtime code imports `CoreCache` directly.
- **Value wedge: Python SDK wrapper + MCP server, not CLI wrapping.** CLI plugins stay as thin shells.
- **One PR per phase.** Each phase lands independently, reviewable, rollback-safe. No big-bang.

## Decisions (locked in by user)

1. Package manager: pip + `pyproject.toml` + lockfile (keep it simple, no new binary prerequisites).
2. MCP: retire legacy `mcp_server.py`; canonical is `infrastructure/mcp_server_2026.py`.
3. Default embedder: `sentence-transformers/all-MiniLM-L6-v2` local (no API key required). OpenAI embeddings adapter optional.
4. Scope for v0.3.0: Phases 0–6. Phase 7 is ADR-only, deferred.

---

## Phase 0 — Guardrails (no behavior change)

**Goal:** make rules enforceable so later phases can't silently regress.

Actions:
- Replace `setup.py` with `pyproject.toml`. Python 3.12+.
- Add `import-linter` contracts matching 2026 §2 layer direction.
- Configure `pytest --cov` thresholds: domain ≥95%, application ≥85%, overall ≥80%.
- Add `pip-audit`, `ruff`, `mypy --strict` on `domain/` and `application/`.
- GitHub Actions workflow: lint → type → import-linter → tests → coverage gate.

Acceptance:
- CI green on main with current code.
- A deliberate `domain → infrastructure` import fails CI.

## Phase 1 — Wire the hex layers

**Goal:** `modern_cli.py` calls use cases. `CoreCache` becomes an internal detail of the storage adapter.

Actions:
- `infrastructure/container.py` composition root; builds `QueryCacheUseCase`, `StoreCacheUseCase`, `InvalidateCacheUseCase`, `CacheMetricsUseCase`.
- Fill missing adapters: `StoragePort` wraps current filesystem cache, `CacheMetricsPort` in-memory + JSON dump, `EventPublisherPort` in-proc log, `QueryNormalizerPort` basic; stub `SemanticIndexPort`/`EmbeddingGeneratorPort`/`TokenCounterPort` with clean `NotImplementedError` until Phase 4/5.
- Rewrite `modern_cli.py` command handlers to call use cases via the container.
- In-memory adapters for every port, used by application-layer tests.

Acceptance:
- `grep CoreCache` shows it imported only from `infrastructure/adapters.py` and tests.
- All existing CLI commands still work; existing tests still pass.
- New application-layer tests cover use cases with in-memory adapters.

## Phase 2 — The SDK wedge (`cached_client`)

**Goal:** ship the thing AI developers actually install.

Actions:
- `src/aicache/integrations/anthropic.py` — `cached_client(Anthropic())` drop-in wrapper around `messages.create`. Streaming + non-streaming. Respects provider-native prompt caching and stacks on top.
- `src/aicache/integrations/openai.py` — same for `chat.completions.create`.
- `TokenCounterPort` adapters: Anthropic SDK count endpoint, `tiktoken` for OpenAI.
- Fixture-recorded unit tests per provider. One gated integration test per provider.
- `docs/SDK_QUICKSTART.md`.

Acceptance:
- `pip install aicache` then `from aicache.integrations.anthropic import cached_client` works.
- Second identical call to wrapped client is a cache hit.

## Phase 3 — MCP server cleanup

**Goal:** one MCP server per bounded context (§3.5).

Actions:
- Promote `infrastructure/mcp_server_2026.py` to canonical; port missing tools/resources from legacy; delete legacy file.
- Tools = writes (store, invalidate); resources = reads (get, stats, metrics).
- Schema + round-trip tests per tool.
- RED metrics per MCP tool.

Acceptance:
- Single MCP server file; schema tests pass; Claude Code can connect and exercise each tool.

## Phase 4 — Semantic caching (the one differentiator)

**Goal:** cross-provider semantic hit, done correctly.

Actions:
- `SemanticIndexPort` adapter: ChromaDB (local). FAISS optional.
- `EmbeddingGeneratorPort` adapter: sentence-transformers local + OpenAI remote optional.
- Wire into existing `QueryCacheUseCase._try_semantic_match`.
- Similarity threshold + confidence scoring with false-positive counter.
- `pip install aicache[semantic]` extras.

Acceptance:
- Integration test: rephrased queries hit cache on the second invocation; semantic metrics separated from exact-match metrics.

## Phase 5 — Observability & cost telemetry (§6)

**Goal:** make the savings story provable.

Actions:
- OpenTelemetry tracing around all use cases, propagated through MCP.
- RED metrics per application use case + MCP tool.
- Structured JSON logs with correlation IDs, PII-masked.
- Per-AI-call log fields: `model`, `model_version`, `prompt_hash`, `tokens_in`, `tokens_out`, `latency_ms`, `cost_usd`, `cache_hit`, `match_type`.
- `aicache stats` shows rolling 30-day savings from those logs.

Acceptance:
- Demo workload produces a structured log stream pipeable into DuckDB for a real `$` number.

## Phase 6 — README reset + killer demo

**Goal:** one demo that proves the wedge.

Actions:
- Rebuild `README.md` around the SDK wedge. Delete feature-list theater.
- `examples/claude_code_agent_demo.py` — repeatable agentic workload, prints before/after costs.
- Collapse `roadmap.md` into three horizons: now, next, later.

Acceptance:
- README readable in 60 seconds with a clear "why install".

## Phase 7 — Rust hot-path kernel (deferred; ADR only)

**Goal:** respect §1 without premature rewrite.

Actions:
- `docs/adr/0001-python-cache-kernel.md` documenting Python kernel now, measured p99, Rust rewrite trigger criteria (e.g. p99 > 10ms at 10k entries).
- No code written in this plan.

## Non-goals

- No web dashboard, team management, federated learning, living brain, GitHub/GitLab/JetBrains integrations, JupyterLab, VSCode, Neovim extensions.
- No TOON expansion beyond what already exists cleanly in hex layers.
- No new CLI plugins beyond claude/openai until end-to-end validation proves value.
